HTML

  • Hypertext Markup Language
  • Frame of webpage
  • The head element contains information about the webpage (not visible to user).
  • The body element represents the visible content shown to the user.
  • The head element is a container for metadata
  • Meta elements contain information about the page that is used by the browser.
  • In semantic HTML elements, the name of the element describes its content and the role it plays on a webpage.
  • When working with visible HTML elements, the content is contained between an opening tag and a closing tag. These tags are most often represented by angle brackets with associated tag names contained inside
  • The alt text is important for screen readers and other assistive technologies. Alt text can also improve search engine rankings for your website.

CSS

  • Cascading Style Sheets
  • Defines style of webpage
  • Makes webpage readable and visually appealing
  • inline CSS
  • direct
  • quick
  • permanent
  • less flexible (each inline style must be edited individually whenever you make a design change)
  • internal CSS style sheet
  • embed an entire stylesheet directly in the HTML file
  • rules only apply to that page
  • you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code
  • a single change to the CSS rule will apply to all tagged page elements
  • external CSS style sheet
  • most common
  • allows developers to keep all their CSS rules in a separate file, which makes design changes easier
  • a "link" element creates the connection to the external file.
  • The "rel" attribute specifies the relationship between the current document and the linked document resource.
  • The "href" attribute specifies the location of the external resource
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

GIT

  • In command line, navigate to main branch of directory
  • Ensure you have latest main - git pull origin main
  • Create a new feature branch - git checkout -b branch-name
  • git status: checks what branch we are currently on
  • add code in VS code and save
  • "Stage your changes," commit changes - git add -A
  • Comments - git commit -m "added code to HTML file"
  • Push changes - git push origin feature/add-html
  • Navigate to repository on GitHub
  • Navigate to branch on GitHub
  • Click "pull requests" tab
  • Click "New Pull Request" button
  • On pull request form, enter title and description
  • Click "create pull request"
  • Click the "Merge pull request" button and confirm

JavaScript

  • Adds interactivity to a webpage
  • Browser reads from top to bottom. Order matters.
  • Variables = containers that store values; start with "let" or "const"
  • JS is case sensitive
  • strings in quotations
  • Boolean = true/false statements
  • Comments can be added with code using /* or //(short). The browser ignores comments.
  • if statements interrupt control flow
  • If one condition is truthy, the computer will execute the code that we provide between curly brackets. But if the condition is not truthy, the code will not execute.
  • Array = single variable that is used to hold a group of data.
  • First item in array has index zero
  • A "for" loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • To use the for loop, we need the shape's index and the array's length
  • x++ increments x by 1 (x+1)
  • Comment out code with /* rather than deleting. Makes code inactive
  • Must call a function. Functions do not automatically execute
  • Use console.log to test interactivity of webpage
